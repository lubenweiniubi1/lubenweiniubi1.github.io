# 05 — 核心模块

> 学习目标：掌握 Node.js 最核心的内置模块——Buffer、Stream、EventEmitter、fs、http。这些是日常开发和面试的核心考点。

---

## 1. Buffer

### 1.1 是什么

Buffer 是 Node.js 处理**二进制数据**的类。JavaScript 原生擅长处理字符串，但网络传输、文件读写、图片视频处理本质都是二进制数据——Buffer 就是为这个场景设计的。

**关键特性：堆外内存。** Buffer 使用 `ArrayBuffer` 在 V8 堆外分配内存，不经过 V8 的 GC 管理。这意味着可以分配远超 V8 堆限制（~1.4GB）的 Buffer，且不会触发频繁 GC。

### 1.2 创建 Buffer

```js
// 推荐方式——自动清零，安全
const buf1 = Buffer.alloc(10);                    // 10 字节，全部归零
const buf2 = Buffer.alloc(10, 'a');               // 10 字节，全部填充 'a'

// 从已有数据创建
const buf3 = Buffer.from('Hello');                // 字符串 → Buffer
const buf4 = Buffer.from([0x48, 0x65, 0x6c]);     // 数组 → Buffer
const buf5 = Buffer.from(buf1);                   // 拷贝另一个 Buffer

// ❌ 不推荐（老 API，可能包含旧内存数据，安全性差）
const buf6 = Buffer.allocUnsafe(10);
```

### 1.3 常用操作

```js
// 编码转换
const buf = Buffer.from('你好', 'utf8');
buf.toString('base64');        // 转为 base64
buf.toString('hex');           // 转为十六进制

// 拼接
const merged = Buffer.concat([buf1, buf2, buf3]);

// 比较
buf1.equals(buf2);             // 内容是否相同

// 遍历
for (const byte of buf) { /* byte 是 0-255 的数字 */ }
```

### 1.4 使用场景

| 场景 | 说明 |
| --- | --- |
| 文件读写 | `fs.readFile()` 不指定编码时返回 Buffer |
| 网络数据 | TCP 流中传输的是 Buffer |
| 编码转换 | `Buffer.from(str, 'utf8').toString('base64')` |
| 图片/文件上传 | multer 等库中 `file.buffer` |
| 加密 | crypto 模块的输入输出都是 Buffer |

## 2. Stream（流）

### 2.1 是什么

Stream 是处理流式数据的抽象接口。核心优势：**内存友好**——不把整个文件加载到内存，而是一块一块处理。

### 2.2 四种类型

| 类型 | 功能 | 例子 |
| --- | --- | --- |
| **Readable** | 可读 | `fs.createReadStream()`、HTTP request |
| **Writable** | 可写 | `fs.createWriteStream()`、HTTP response |
| **Duplex** | 双工（可读可写） | `net.Socket` |
| **Transform** | 转换（读写之间做变换） | `zlib.createGzip()`、`crypto.createCipher()` |

### 2.3 pipe 和背压（Backpressure）

```js
const fs = require('fs');
const zlib = require('zlib');

// 管道：读 → 压缩 → 写
fs.createReadStream('input.txt')       // Readable
  .pipe(zlib.createGzip())             // Transform
  .pipe(fs.createWriteStream('output.txt.gz')); // Writable
```

**背压问题：** 当可读流速度快于可写流时，数据在内存中堆积。

**Node.js 的背压处理：**
1. 可写流的 `write()` 返回 `false` 时，表示内部缓冲区已满
2. pipe 机制自动暂停可读流
3. 可写流发出 `drain` 事件后，恢复可读流

```js
// 手动处理背压
function copy(src, dest) {
  const rs = fs.createReadStream(src);
  const ws = fs.createWriteStream(dest);

  rs.on('data', chunk => {
    const canContinue = ws.write(chunk);
    if (!canContinue) {          // 写满了
      rs.pause();                // 暂停读取
      ws.once('drain', () => rs.resume()); // 排空后恢复
    }
  });
}
```

### 2.4 常用 Stream 模式

```js
// flowing 模式 —— 自动推送数据
rs.on('data', chunk => { /* ... */ });
rs.on('end', () => console.log('done'));

// paused 模式 —— 手动拉取数据
rs.on('readable', () => {
  let chunk;
  while ((chunk = rs.read()) !== null) {
    // 处理 chunk
  }
});
```

## 3. EventEmitter

### 3.1 是什么

`events` 模块导出的类，Node.js 事件驱动的核心实现。**发布-订阅模式**：有人注册监听（on），有人触发事件（emit）。

```js
const EventEmitter = require('events');

class MyService extends EventEmitter {
  start() {
    this.emit('started', { time: Date.now() });
  }
}

const service = new MyService();
service.on('started', info => console.log('Started at', info.time));
service.start();
```

**Node.js 中大部分能发事件的模块都继承自 EventEmitter**：http.Server、fs.ReadStream、net.Socket、process……

### 3.2 核心 API

| 方法 | 作用 |
| --- | --- |
| `on(event, listener)` | 注册监听器 |
| `emit(event, ...args)` | 触发事件 |
| `once(event, listener)` | 一次性监听（触发后自动移除） |
| `off(event, listener)` | 移除监听器 |
| `removeAllListeners([event])` | 移除所有监听器 |
| `listenerCount(event)` | 返回监听器数量 |
| `eventNames()` | 返回所有已注册的事件名 |

### 3.3 关键细节

**同步执行：** `emit` 是同步的——所有监听器在当前 tick 依次执行。

**错误事件：** `error` 事件是特殊的——如果触发了 `error` 事件但没有监听器，会**抛出错误导致进程崩溃**。

**监听器上限：** 默认单事件最多 10 个监听器——`EventEmitter.defaultMaxListeners`。超过会打印 `MaxListenersExceededWarning`。用 `emitter.setMaxListeners(n)` 调大，**但如果需要调大，通常意味着设计有问题**——检查是否存在内存泄漏。

### 3.4 手动实现

面试高频手写题：

```js
class EventEmitter {
  constructor() {
    this._events = Object.create(null);
  }

  on(event, listener) {
    (this._events[event] ||= []).push(listener);
    return this;
  }

  emit(event, ...args) {
    const listeners = this._events[event];
    if (!listeners || listeners.length === 0) return false;
    listeners.forEach(fn => fn(...args));  // 同步执行
    return true;
  }

  once(event, listener) {
    const wrapper = (...args) => {
      this.off(event, wrapper);  // 触发前先移除
      listener(...args);
    };
    return this.on(event, wrapper);
  }

  off(event, listener) {
    const listeners = this._events[event];
    if (!listeners) return this;
    this._events[event] = listeners.filter(fn => fn !== listener);
    return this;
  }
}
```

## 4. fs 模块

### 4.1 四套 API

| 风格 | 示例 | 特点 | 适用场景 |
| --- | --- | --- | --- |
| **同步** | `fs.readFileSync(path)` | 阻塞事件循环 | CLI、初始化脚本 |
| **回调** | `fs.readFile(path, cb)` | 传统异步 | 旧代码兼容 |
| **Promise** | `fs.promises.readFile(path)` | 现代异步，推荐 | 新代码主要方式 |
| **Stream** | `fs.createReadStream(path)` | 分块处理 | 大文件 |

### 4.2 选择原则

```
小文件          → Promise API 或 readFileSync（CLI only）
大文件          → Stream
生产环境服务端   → 禁止使用 Sync（会阻塞事件循环）
```

Node 20+ 的 `fs/promises` 简写：
```js
const fs = require('fs/promises');  // Node 20+ 直接可用
const data = await fs.readFile('file.txt', 'utf8');
```

### 4.3 常用操作

```js
const fs = require('fs/promises');
const { constants } = require('fs');

// 读写
await fs.readFile('a.txt', 'utf8');
await fs.writeFile('b.txt', 'content');

// 追加
await fs.appendFile('log.txt', 'new line\n');

// 目录
await fs.mkdir('new-dir', { recursive: true });
const files = await fs.readdir('some-dir');

// 状态
const stat = await fs.stat('file.txt');
stat.isFile();      // 是文件？
stat.isDirectory(); // 是目录？
stat.size;          // 字节数

// 权限检查
await fs.access('file.txt', constants.R_OK | constants.W_OK);

// 删除
await fs.unlink('file.txt');           // 删文件
await fs.rmdir('dir');                 // 删空目录
await fs.rm('dir', { recursive: true }); // 递归删除（Node 14+）
```

## 5. http 模块

### 5.1 创建 HTTP 服务

```js
const http = require('http');

const server = http.createServer((req, res) => {
  // req: IncomingMessage（Readable Stream）
  // res: ServerResponse（Writable Stream）

  res.writeHead(200, { 'Content-Type': 'application/json' });
  res.end(JSON.stringify({ ok: true }));
});

server.listen(3000, () => {
  console.log('Server running on http://localhost:3000');
});
```

### 5.2 请求处理流程

```
客户端发起请求
    │
    ▼
Node.js net 模块建立 TCP 连接
    │
    ▼
HTTP parser 解析请求报文
    ├── 构建 IncomingMessage（req）
    └── 构建 ServerResponse（res）
    │
    ▼
触发 'request' 事件 → createServer 的回调被调用
    │
    ▼
业务代码通过 res.writeHead() / res.end() 返回响应
```

### 5.3 `req` 和 `res` 的本质

- **`req`（IncomingMessage）**：Readable Stream。你可以 `req.on('data')` 读请求体，`req.headers` 读头信息
- **`res`（ServerResponse）**：Writable Stream。你可以 `res.write()` 写响应体，`res.end()` 结束

这也是为什么可以直接 pipe：

```js
http.createServer((req, res) => {
  fs.createReadStream('big-file.zip').pipe(res);
});
```

### 5.4 获取请求体

```js
// JSON body 解析
function parseBody(req) {
  return new Promise((resolve, reject) => {
    const chunks = [];
    req.on('data', chunk => chunks.push(chunk));
    req.on('end', () => {
      try {
        resolve(JSON.parse(Buffer.concat(chunks).toString()));
      } catch (e) {
        reject(e);
      }
    });
    req.on('error', reject);
  });
}
```

大多数开发者不会手写这些，而是用 Express 的 `express.json()` 或 Koa 的 `koa-body` 中间件。

## 6. 快速回顾

1. **Buffer**：二进制数据容器，堆外内存，编码转换
2. **Stream**：四种类型，pipe + 背压 + drain，内存友好
3. **EventEmitter**：发布-订阅，on/emit/once/off，once 是包装函数自删
4. **fs**：四套 API，服务端禁用 Sync，大文件用 Stream
5. **http**：req = Readable，res = Writable，pipe 是自然的
