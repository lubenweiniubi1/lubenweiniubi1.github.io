# 07 — 多进程与部署

> 学习目标：理解 Node.js 如何利用多核 CPU、进程间通信、以及生产环境的部署方案。

---

## 1. 为什么需要多进程

Node.js 的 JS 主线程是**单线程**的，一个进程只能跑在一个 CPU 核心上。

```
8 核服务器 → 直接跑 Node.js → 只用 1 个核，剩下 7 个闲置
```

解决方案：多进程。让每个 CPU 核心跑一个 Node.js 进程。

## 2. cluster 模块

### 2.1 原理

```
                ┌─────────────┐
    请求进来 →   │  Master 进程 │  ← 负责分发连接
                └──┬───┬───┬──┘
                   │   │   │
          ┌────────┘   │   └────────┐
          ▼            ▼            ▼
    ┌──────────┐ ┌──────────┐ ┌──────────┐
    │ Worker 0 │ │ Worker 1 │ │ Worker 2 │  ← 实际处理请求
    │ CPU 0    │ │ CPU 1    │ │ CPU 2    │
    └──────────┘ └──────────┘ └──────────┘
```

- Master 负责 `accept` 连接，通过 **Round-robin（轮询）** 分发给 Worker
- 每个 Worker 是独立进程，有独立的 V8 实例和 Event Loop
- 所有 Worker **共享同一个端口**

### 2.2 基本用法

```js
const cluster = require('cluster');
const http = require('http');
const numCPUs = require('os').cpus().length;

if (cluster.isMaster) {
  // Master：fork 多个 Worker
  console.log(`Master ${process.pid} is running`);

  for (let i = 0; i < numCPUs; i++) {
    cluster.fork();
  }

  cluster.on('exit', (worker, code, signal) => {
    console.log(`Worker ${worker.process.pid} died`);
    cluster.fork(); // 挂了自动重启
  });
} else {
  // Worker：处理请求
  http.createServer((req, res) => {
    res.writeHead(200);
    res.end(`Handled by Worker ${process.pid}\n`);
  }).listen(8000);

  console.log(`Worker ${process.pid} started`);
}
```

### 2.3 IPC 通信

```js
// Master → Worker
worker.send({ cmd: 'shutdown' });

// Worker → Master
process.on('message', msg => {
  if (msg.cmd === 'shutdown') {
    server.close(() => process.exit(0));
  }
});

// Worker → Master
process.send({ cmd: 'notify', data: 'done' });
```

## 3. PM2

### 3.1 PM2 是什么

PM2 是基于 cluster 的**生产级进程管理器**，封装了 cluster 的所有细节，提供开箱即用的多进程方案。

### 3.2 核心功能

| 功能 | 说明 | 命令 |
| --- | --- | --- |
| 多进程 | 按 CPU 核心数启动 | `pm2 start app.js -i max` |
| 进程守护 | 挂了自动重启 | 默认开启 |
| 零停机重载 | 逐个重启 Worker，不丢请求 | `pm2 reload app` |
| 日志管理 | 自动收集 stdout/stderr | `pm2 logs` |
| 开机自启 | 系统重启后自动启动 | `pm2 startup` |
| 监控 | CPU、内存、请求频率 | `pm2 monit` |

### 3.3 常用命令

```bash
pm2 start app.js              # 启动
pm2 start app.js -i max       # 按 CPU 核心数启动多实例
pm2 start app.js --name api   # 指定名称
pm2 list                      # 列出所有进程
pm2 logs                      # 查看日志
pm2 logs api                  # 某个进程的日志
pm2 reload api                # 零停机重载
pm2 stop api                  # 停止
pm2 delete api                # 删除
pm2 save                      # 保存当前进程列表（配合 startup）
pm2 startup                   # 设置开机自启
```

### 3.4 配置文件

```js
// ecosystem.config.js
module.exports = {
  apps: [{
    name: 'api',
    script: './src/app.js',
    instances: 'max',          // 按 CPU 核心数
    exec_mode: 'cluster',      // 集群模式
    env: {
      NODE_ENV: 'production',
      PORT: 3000,
    },
    error_file: './logs/err.log',
    out_file: './logs/out.log',
    merge_logs: true,
    max_memory_restart: '500M', // 内存超限自动重启
  }]
};
```

```bash
pm2 start ecosystem.config.js
```

## 4. child_process

### 4.1 四种方法对比

| 方法 | 返回 | 适用场景 | 注意 |
| --- | --- | --- | --- |
| `spawn` | Stream | 长时间运行、大量输出的命令 | 流式处理，不积压内存 |
| `exec` | Buffer | 简单短命令 | **有 maxBuffer 限制（默认 1MB）** |
| `execFile` | Buffer | 执行可执行文件 | 不启动 shell，更安全 |
| `fork` | ChildProcess | 执行 Node.js 模块 | **自带 IPC 通信通道** |

### 4.2 使用示例

```js
const { spawn, exec, fork } = require('child_process');

// spawn —— 流式输出，适合长时间任务
const child = spawn('ffmpeg', ['-i', 'input.mp4', 'output.avi']);
child.stdout.on('data', data => console.log(`stdout: ${data}`));
child.stderr.on('data', data => console.error(`stderr: ${data}`));
child.on('close', code => console.log(`exited with code ${code}`));

// exec —— Buffer 输出，适合短命令
exec('ls -la', { maxBuffer: 1024 * 1024 * 10 }, (err, stdout, stderr) => {
  if (err) return console.error(err);
  console.log(stdout);
});

// fork —— Node 脚本，自带 IPC
const worker = fork('./worker.js');
worker.send({ task: 'process', data: largeArray });
worker.on('message', result => console.log('Result:', result));
```

### 4.3 选择策略

```
需要调用其他 Node 脚本？ → fork（自带 IPC）
需要执行长时间命令、大量输出？ → spawn（Stream 不积压内存）
需要执行简单 shell 命令？ → exec
需要执行可执行文件？ → execFile（更安全）
```

## 5. Worker Threads

Node.js 12+ 支持的多线程方案，和 cluster 的区别：

| | Worker Threads | cluster |
| --- | --- | --- |
| 粒度 | 线程 | 进程 |
| 内存 | **共享** `SharedArrayBuffer` | 独立，不共享 |
| 通信 | `postMessage` / MessageChannel | IPC（进程间通信） |
| 适用 | CPU 密集型计算 | I/O 密集型（充分利用多核） |

```js
const { Worker, isMainThread, parentPort } = require('worker_threads');

if (isMainThread) {
  const worker = new Worker(__filename);
  worker.on('message', msg => console.log('Result:', msg));
  worker.postMessage({ num: 42 });
} else {
  parentPort.on('message', msg => {
    // 在子线程中执行 CPU 密集型计算
    const result = fibonacci(msg.num);
    parentPort.postMessage(result);
  });
}
```

## 6. 进程间通信（IPC）

### 6.1 方案对比

| 方案 | 适用场景 | 复杂度 |
| --- | --- | --- |
| `fork` 自带 IPC | 父子进程 | 低 |
| Redis Pub/Sub | 所有进程（跨机器） | 中 |
| Redis List / Stream | 任务队列 | 中 |
| RabbitMQ / Kafka | 企业级消息 | 高 |
| 文件 / 数据库 | 简单状态共享 | 低（性能差） |

### 6.2 Redis Pub/Sub 示例

```js
// publisher.js
const redis = require('redis');
const pub = redis.createClient();
pub.publish('channel:order', JSON.stringify({ orderId: 123 }));

// subscriber.js
const sub = redis.createClient();
sub.subscribe('channel:order');
sub.on('message', (channel, message) => {
  console.log(JSON.parse(message));
});
```

## 7. CLI 工具开发

### 7.1 最小结构

```
my-cli/
├── package.json    # 有 bin 字段
├── bin/
│   └── cli.js      # 入口文件，有 shebang
└── src/
    └── index.js    # 业务逻辑
```

**package.json：**
```json
{
  "name": "my-cli",
  "bin": { "mycli": "./bin/cli.js" }
}
```

**bin/cli.js：**
```js
#!/usr/bin/env node
const args = process.argv.slice(2);
console.log(`Hello, ${args[0] || 'world'}!`);
```

### 7.2 常用库

| 库 | 用途 |
| --- | --- |
| `commander` | 命令行参数解析（子命令、选项） |
| `inquirer` | 交互式问答（选择、输入、确认） |
| `chalk` | 彩色终端输出 |
| `ora` | 加载 spinner 动画 |
| `boxen` | 终端盒子/边框 |

### 7.3 开发流程

```bash
npm link          # 本地测试——将包链接到全局
mycli hello       # 直接测试命令
npm publish       # 发布后别人 npm i -g 就能用
```

## 8. 快速回顾

1. cluster：Master fork Worker，共享端口，Round-robin 分发
2. PM2：cluster 的生产级封装，进程守护 + 零停机重载 + 日志
3. child_process：spawn（Stream）/ exec（Buffer）/ fork（IPC）/ execFile（无 shell）
4. Worker Threads：多线程，可共享内存，适合 CPU 密集型
5. 进程间通信：fork 自带 IPC；跨进程用 Redis Pub/Sub
6. CLI：shebang + bin 字段 + npm link + commander + chalk
