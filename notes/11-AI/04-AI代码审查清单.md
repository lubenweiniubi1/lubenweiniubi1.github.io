# AI 生成代码 — 通用前端审查清单

> 每次 AI 生成代码后，按此清单逐项审查，不因代码来源而降低标准。

---

## 一、类型安全 ✅

- [ ] `any` 是否都用具体类型替换了
- [ ] Props / 函数参数都有完整类型定义
- [ ] API 返回值有对应的 interface/type
- [ ] 泛型使用正确，没有不必要的类型断言 `as`
- [ ] 没有 `// @ts-ignore` 或 `as any` 逃避检查

---

## 二、React 专项 ✅

- [ ] `useEffect` 依赖数组完整且正确
- [ ] 没有在 effect 中遗漏 cleanup（定时器、订阅、事件监听）
- [ ] `useMemo` / `useCallback` 合理使用，避免过早优化
- [ ] 列表渲染有稳定的、唯一的 `key`
- [ ] 没有在渲染中直接修改 state
- [ ] 条件渲染处理了 `null` / `undefined` / 空数组
- [ ] 组件拆分粒度合理（单一职责，不超过 ~200 行）

---

## 三、性能 ✅

- [ ] 没有在循环/条件中调用 hooks
- [ ] 没有将新对象/函数直接作为依赖导致无限重渲染
- [ ] 大列表考虑虚拟滚动（超过 100 项）
- [ ] 图片使用懒加载 `loading="lazy"`
- [ ] 事件处理函数无防抖/节流的必要场景已补上
- [ ] 没有不必要的 `useEffect`（可以用事件处理器替代）
- [ ] 导入使用按需引入（`import { debounce } from 'lodash/debounce'` 而非整个库）

---

## 四、安全 ✅

- [ ] 没有 `dangerouslySetInnerHTML`，如有则确认已做 XSS 消毒（DOMPurify）
- [ ] 用户输入有前端校验（必填、格式、长度）
- [ ] 没有硬编码的 API key / token / 密码 → 走环境变量
- [ ] URL 参数/用户输入没有直接拼接到 SQL/命令/HTML 中
- [ ] 外部链接使用 `rel="noopener noreferrer"`

---

## 五、边界条件 ✅

- [ ] Loading 状态有处理
- [ ] Error 状态有处理（try/catch、error boundary）
- [ ] 空数据状态有处理（空列表提示、empty state）
- [ ] 网络异常有处理（超时、断网）
- [ ] 极端值有处理（0、空字符串、负数、超长文本）
- [ ] 组件卸载后的异步回调不会 setState（竞态条件）

---

## 六、可访问性 a11y ✅

- [ ] 可点击元素用 `<button>`，不用 `<div onclick>`
- [ ] 图标按钮有 `aria-label`
- [ ] 表单有 `<label>` 关联
- [ ] 图片有 `alt` 属性（装饰性图片用 `alt=""`）
- [ ] 键盘可操作（Enter、Esc、Tab 顺序合理）
- [ ] 焦点管理正确（弹窗打开聚焦、关闭归还）

---

## 七、CSS / 样式 ✅

- [ ] 没有用 `!important`，如有则说明原因
- [ ] z-index 在合理范围（< 1000），不随意设定
- [ ] 没有全局样式污染（已使用 CSS Modules / BEM / scoped）
- [ ] 响应式断点合理，没有断点遗漏
- [ ] 使用 CSS 变量/design tokens 而非硬编码色值

---

## 八、代码规范 ✅

- [ ] 命名清晰有意义（没有 `data`、`item`、`val` 等模糊命名）
- [ ] 函数单一职责，不超过 ~30 行
- [ ] 没有注释掉的死代码
- [ ] 没有 `console.log` 残留
- [ ] 魔法数字已提取为常量
- [ ] 导入顺序合理（第三方 > 内部模块 > 相对路径）

---

## 九、测试思维 ✅

- [ ] **核心逻辑**：工具函数/自定义 hooks 需要单元测试
- [ ] **关键流程**：用户主流程需要集成测试
- [ ] 测试覆盖了正常路径 + 异常路径 + 边界值
- [ ] Mock 合理，没有过度 mock 导致测试失去意义

---

## 十、兼容性 ✅

- [ ] 确认目标浏览器范围（browserslist）
- [ ] CSS 新特性有 fallback 或确认 Autoprefixer 覆盖
- [ ] JS 新 API 确认 babel/polyfill 已覆盖
- [ ] 移动端 Safari 特殊处理（100vh 问题、日期格式等）

---

## 快速命令

```bash
# 类型检查
npx tsc --noEmit

# 代码规范检查
npx eslint <file> --fix

# 格式化
npx prettier --write <file>
```

---

> **原则**：AI 生成的代码和手写代码适用完全相同的标准。能通过这个 checklist 的代码，才算可交付。
