========================================
       React Fiber 学习资源推荐
========================================

推荐学习路径（从浅到深）
========================================

一、先理解 Why：React 为什么需要 Fiber
----------------------------------------
在深入 Fiber 之前，先搞清楚它解决什么问题：

  React Fiber Architecture
  https://github.com/acdlite/react-fiber-architecture
  - Andrew Clark（React 核心团队）写的官方介绍。必读！
    这是理解 Fiber 动机最好的入口，解释了 Stack Reconciler
    的问题和 Fiber 的解决方案。


二、逐步深入原理
----------------------------------------
1. Lin Clark - A Cartoon Intro to Fiber（视频）
   https://www.youtube.com/watch?v=ZCuYPiUIONs
   - 用动画解释 Fiber，非常适合建立直觉。
     先在脑子里形成"画面感"，再看代码就不会那么抽象。

2. Inside Fiber: in-depth overview of the new reconciliation
   algorithm in React
   https://indepth.dev/posts/1008/inside-fiber-in-depth-overview-of-the-new-reconciliation-algorithm-in-react
   - Max Koretskyi 的系列文章，把 Fiber 的每个阶段
    （render phase、commit phase）拆得很细。

3. 卡颂的《React 技术揭秘》
   - 国内最推荐的 React 源码解读资料，中文、有配套网站和视频，
     从理念到源码逐行讲解，非常适合跟着跑一遍。


三、动手：用最小实现来理解
----------------------------------------
1. build-your-own-react
   https://pomb.us/build-your-own-react/
   - Rodrigo Pombo 的经典教程，用约 200 行代码手写一个
     迷你 React（包含简化的 Fiber 调度）。
     强烈推荐跟着敲一遍，敲完你对 Fiber 的理解会从
    "好像懂了"变成"真的懂了"。

2. didact（配套仓库）
   https://github.com/pomber/didact
   - 上面教程对应的代码仓库。


四、高阶：调度和 Lane 模型
----------------------------------------
  React Internals Deep Dive
  https://jser.dev/react/
  - JSer 的系列文章，深入 lane 模型、优先级调度、
    Suspense 的内部实现。


========================================
学习建议
========================================

  阶段      目标                         推荐资源
  ------    ------                       ------
  直觉      知道 Fiber 是什么，为什么需要   A Cartoon Intro to Fiber（视频）
  概念      理解 Fiber Node、双缓冲、       acdlite 的 Fiber Architecture 文档
            Render/Commit 阶段
  源码      跟着走一遍核心流程             卡颂《React 技术揭秘》
  内化      能自己写出简化版               build-your-own-react 动手教程
  精通      Lane 模型、优先级、中断恢复     JSer 系列


========================================
关键提醒
========================================

  关键是不要只看不动手。Fiber 是工程上的设计，看十遍不如自己
  写一遍 workLoop。跟着 build-your-own-react 敲完，你会发现
  那些碎片化的概念自动就连成了一张网。
