# 📚 JavaScript函数式编程深度教学

函数式编程（Functional Programming, FP）是一种强大的编程范式，它将计算视为数学函数的求值过程。在JavaScript中，函数式编程能够显著提升代码的**可维护性**、**可测试性**和**可复用性**。

---

## 一、函数式编程核心理念

### 1.1 什么是函数式编程？

函数式编程是一种**声明式**的编程范式，强调：
- ✅ **纯函数**：相同输入始终得到相同输出
- ✅ **不可变性**：数据一旦创建就不能被修改
- ✅ **函数组合**：通过组合简单函数构建复杂功能
- ✅ **避免副作用**：函数不应该改变外部状态

### 1.2 与面向对象编程对比

| 特性 | 函数式编程 | 面向对象编程 |
|------|-----------|-------------|
| 数据存储 | 不可变数据结构 | 可变对象状态 |
| 核心构建单元 | 函数 | 类与对象 |
| 控制流 | 函数组合与递归 | 循环与条件语句 |
| 状态管理 | 显式传递 | 隐式封装 |

## 二、核心概念详解

### 2.1 纯函数（Pure Function）

**定义**：纯函数是不依赖外部状态，也不改变任何外部状态的函数。对于相同的输入，纯函数总是返回相同的输出。

```javascript
// ✅ 纯函数示例
const add = (a, b) => a + b;
const multiply = (x, y) => x * y;

// ❌ 非纯函数示例（依赖外部变量）
let base = 10;
const impureAdd = (num) => base + num; // 依赖外部变量base

// ❌ 非纯函数示例（产生副作用）
const logAndAdd = (a, b) => {
    console.log(`Adding ${a} and ${b}`); // 副作用：修改外部状态
    return a + b;
};
```

**纯函数的优势**：
- 🎯 **可预测性**：相同输入→相同输出
- 🧪 **易于测试**：无需设置复杂的测试环境
- 🔄 **天然适合并发**：无共享状态冲突
- 🐛 **易于调试**：函数行为完全由输入决定

### 2.2 不可变性（Immutability）

不可变性意味着数据一旦创建就不能被修改，任何"修改"操作都会返回新的数据。

```javascript
// ❌ 可变方式（修改原数组）
const arr1 = [1, 2, 3];
arr1.push(4); // 直接修改原数组
console.log(arr1); // [1, 2, 3, 4]

// ✅ 不可变方式（返回新数组）
const arr2 = [1, 2, 3];
const newArr = [...arr2, 4]; // 创建新数组
console.log(arr2); // [1, 2, 3]
console.log(newArr); // [1, 2, 3, 4]

// 使用数组方法实现不可变性
const users = [
    { id: 1, name: '张三', active: true },
    { id: 2, name: '李四', active: false }
];

// ❌ 可变修改
users[0].name = '王五'; // 直接修改对象

// ✅ 不可变修改
const updatedUsers = users.map(user => 
    user.id === 1 ? { ...user, name: '王五' } : user
);
```

**工具库推荐**：
- **Immer**：让你用可变的语法编写不可变的代码
- **Immutable.js**：Facebook出品的不可变数据结构库

### 2.3 高阶函数（Higher-Order Function）

高阶函数是接受函数作为参数或返回函数的函数。

```javascript
// 接受函数作为参数
const map = (array, fn) => {
    const result = [];
    for (let i = 0; i < array.length; i++) {
        result.push(fn(array[i], i, array));
    }
    return result;
};

// 返回函数
const createMultiplier = (factor) => {
    return (num) => num * factor;
};

const double = createMultiplier(2);
console.log(double(5)); // 10

// 常用高阶函数
const numbers = [1, 2, 3, 4, 5];

// map：转换数组元素
const doubled = numbers.map(n => n * 2); // [2, 4, 6, 8, 10]

// filter：筛选数组元素
const evens = numbers.filter(n => n % 2 === 0); // [2, 4]

// reduce：聚合数组元素
const sum = numbers.reduce((acc, n) => acc + n, 0); // 15
```

### 2.4 柯里化（Currying）

柯里化是将一个多参数函数转换为一系列单参数函数的技术。

```javascript
// 手动柯里化
const add = (a) => (b) => (c) => a + b + c;
console.log(add(1)(2)(3)); // 6

// 通用柯里化函数实现
function curry(fn) {
    return function curried(...args) {
        if (args.length >= fn.length) {
            return fn.apply(this, args);
        } else {
            return function(...nextArgs) {
                return curried.apply(this, args.concat(nextArgs));
            };
        }
    };
}

// 使用示例
const addThree = curry((a, b, c) => a + b + c);
console.log(addThree(1)(2)(3)); // 6
console.log(addThree(1, 2)(3)); // 6
console.log(addThree(1, 2, 3)); // 6

// 实际应用场景：表单验证
const validate = curry((rule, value) => rule.test(value));

const isEmail = validate(/^[^\s@]+@[^\s@]+\.[^\s@]+$/);
const isPhone = validate(/^\d{11}$/);

console.log(isEmail('test@example.com')); // true
console.log(isPhone('13800138000')); // true
```

**柯里化的优势**：
- 🔧 **参数复用**：可以预设部分参数
- ⏸️ **延迟执行**：可以分多次传入参数
- 🧩 **函数组合**：更容易组合函数

### 2.5 函数组合（Function Composition）

函数组合是将多个函数串联执行，形成数据处理管道。

```javascript
// 从右到左组合（compose）
const compose = (...fns) => (x) => 
    fns.reduceRight((acc, fn) => fn(acc), x);

// 从左到右组合（pipe）
const pipe = (...fns) => (x) => 
    fns.reduce((acc, fn) => fn(acc), x);

// 示例函数
const add1 = (x) => x + 1;
const multiply2 = (x) => x * 2;
const square = (x) => x * x;

// 使用compose（从右到左）
const result1 = compose(square, multiply2, add1)(3);
// 执行顺序：add1(3) -> 4, multiply2(4) -> 8, square(8) -> 64
console.log(result1); // 64

// 使用pipe（从左到右）
const result2 = pipe(add1, multiply2, square)(3);
// 执行顺序：add1(3) -> 4, multiply2(4) -> 8, square(8) -> 64
console.log(result2); // 64
```

## 三、实战案例

### 3.1 数据处理管道

```javascript
// 数据清洗和转换管道
const users = [
    { id: 1, name: '  张三  ', email: 'zhangsan@example.com', age: 25, active: true },
    { id: 2, name: '  李四  ', email: 'invalid-email', age: 30, active: false },
    { id: 3, name: '  王五  ', email: 'wangwu@example.com', age: 28, active: true },
    { id: 4, name: '  赵六  ', email: 'zhaoliu@example.com', age: 22, active: true }
];

// 辅助函数
const trim = (str) => str.trim();
const toUpperCase = (str) => str.toUpperCase();
const isValidEmail = (email) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
const isActive = (user) => user.active;
const getName = (user) => user.name;
const getEmail = (user) => user.email;

// 数据处理管道
const processUsers = pipe(
    // 筛选活跃用户
    (users) => users.filter(isActive),
    // 清洗姓名
    (users) => users.map(user => ({
        ...user,
        name: pipe(trim, toUpperCase)(user.name)
    })),
    // 筛选有效邮箱
    (users) => users.filter(user => isValidEmail(user.email)),
    // 提取姓名
    (users) => users.map(getName)
);

const result = processUsers(users);
console.log(result); // ['张三', '王五', '赵六']
```

 
### 3.3 表单验证系统

```javascript
// 验证器组合
const isRequired = (value) => value !== null && value !== undefined && value !== '';
const minLength = (len) => (value) => value.length >= len;
const maxLength = (len) => (value) => value.length <= len;
const isEmail = (value) => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value);

// 验证器组合器
const all = (...validators) => (value) => 
    validators.every(validator => validator(value));

const any = (...validators) => (value) => 
    validators.some(validator => validator(value));

// 创建验证规则
const validateUsername = all(
    isRequired,
    minLength(3),
    maxLength(20)
);

const validatePassword = all(
    isRequired,
    minLength(8)
);

const validateEmail = all(
    isRequired,
    isEmail
);

// 使用验证器
console.log(validateUsername('ab')); // false（长度不足）
console.log(validateUsername('abc')); // true
console.log(validateEmail('test@example.com')); // true
console.log(validateEmail('invalid')); // false
```

### 3.4 异步数据处理

```javascript
// 异步函数组合
const fetchUser = async (userId) => {
    // 模拟API调用
    return new Promise(resolve => 
        setTimeout(() => resolve({ id: userId, name: `用户${userId}` }), 100)
    );
};

const fetchPosts = async (userId) => {
    // 模拟API调用
    return new Promise(resolve => 
        setTimeout(() => resolve([
            { id: 1, title: '文章1', authorId: userId },
            { id: 2, title: '文章2', authorId: userId }
        ]), 100)
    );
};

// 异步管道
const asyncPipe = (...fns) => async (x) => {
    let result = x;
    for (const fn of fns) {
        result = await fn(result);
    }
    return result;
};

// 使用示例
const getUserPosts = asyncPipe(
    fetchUser,
    (user) => fetchPosts(user.id),
    (posts) => posts.filter(post => post.title.includes('文章'))
);

getUserPosts(1).then(posts => console.log(posts));
```

 ## 五、最佳实践与注意事项

### 5.1 何时使用函数式编程？

✅ **适合场景**：
- 数据转换和处理
- 表单验证
- 配置管理
- 纯计算逻辑
- 测试驱动开发

❌ **不适合场景**：
- 需要频繁修改状态的UI交互
- 性能敏感的循环操作
- 需要大量副作用的操作（如DOM操作）

## 六、总结

函数式编程不是银弹，但它提供了一套强大的工具和思维方式：

🎯 **核心价值**：
- 代码更易测试和维护
- 减少意外的副作用
- 提高代码复用性
- 更好的并发支持

🚀 **学习路径**：
1. 掌握纯函数和不可变性
2. 熟练使用高阶函数（map/filter/reduce）
3. 理解柯里化和函数组合
4. 学习函子和Monad等高级概念
5. 在实际项目中渐进式应用
