## 📋 核心执行顺序规则

### 基本原则
```
原生捕获 → 原生冒泡 → React捕获 → React冒泡 
```

**关键原因**：React的所有事件（包括捕获和冒泡）都是在原生事件冒泡到根节点后才开始处理的！

---

## 🧪 测试场景1：父子嵌套 + 完整事件流

### 代码示例
```javascript
import React, { useEffect } from 'react';

function App() {
  useEffect(() => {
    // 原生事件绑定
    const parent = document.getElementById('parent');
    const child = document.getElementById('child');

    // 原生捕获
    parent.addEventListener('click', () => {
      console.log('原生-父捕获');
    }, true);

    child.addEventListener('click', () => {
      console.log('原生-子捕获');
    }, true);

    // 原生冒泡
    parent.addEventListener('click', () => {
      console.log('原生-父冒泡');
    }, false);

    child.addEventListener('click', () => {
      console.log('原生-子冒泡');
    }, false);
  }, []);

  return (
    <div id="parent" 
         onClickCapture={() => console.log('React-父捕获')}
         onClick={() => console.log('React-父冒泡')}>
      父元素
      <div id="child" 
           onClickCapture={() => console.log('React-子捕获')}
           onClick={() => console.log('React-子冒泡')}>
        子元素
      </div>
    </div>
  );
}
```

### 点击子元素时的执行顺序
```
原生-父捕获
原生-子捕获
原生-子冒泡
原生-父冒泡

原生-父捕获
原生-子捕获
原生-子冒泡
原生-父冒泡
React-父捕获
React-子捕获
React-子冒泡
React-父冒泡
```

**注意**：React 17+ 版本中，React事件会插入到原生冒泡之前！

## 🧪 测试场景2：React 16 vs React 17+ 的差异

### React 16 的执行顺序（根节点在document）
```javascript
// 点击子元素
原生-父捕获
原生-子捕获
原生-子冒泡
原生-父冒泡
React-父捕获
React-子捕获
React-子冒泡
React-父冒泡
```

### React 17+ 的执行顺序（根节点在root container）
```javascript
// 点击子元素
React-父捕获
React-子捕获
原生-父捕获
原生-子捕获
原生-子冒泡
原生-父冒泡
React-子冒泡
React-父冒泡
```

**关键区别**：React 17+ 将事件委托从 `document` 移到了 root 容器，所以 React 事件会在原生冒泡到达 root 容器时执行，而不是等到冒泡到 document！

## 🧪 测试场景4：阻止冒泡的影响

### 代码示例
```javascript
import React, { useEffect } from 'react';

function App3() {
  useEffect(() => {
    const child = document.getElementById('child');

    child.addEventListener(
      'click',
      (e) => {
        console.log('原生捕获');
        e.stopPropagation(); // 阻止原生冒泡
      },
      true,
    );
  }, []);

  return (
    <div onClick={() => console.log('React-父冒泡')} onClickCapture={() => console.log('React-父捕获')}>
      父元素
      <div id='child' onClick={() => console.log('React-子冒泡')} onClickCapture={() => console.log('React-子捕获')}>
        子元素
      </div>
    </div>
  );
}
```

### 执行结果

```
React 16:
原生捕获

React 17:
React-父捕获
React-子捕获
原生捕获
```

# 为什么16 和 17 打印顺序不一样

你的理解是对的，React 17 把事件委托从 `document` 移到了 `#root` 容器上。**捕获提前的原因很简单：`#root` 在 DOM 树中位于 `#parent` 之上。**

## 事件传播路径分析

你的 DOM 结构是：

```
document → html → body → #root → #parent → #child
```

React 17 在 `#root` 上注册了**两个**原生监听器：
- **捕获阶段监听器**：处理所有 React 的 `onClickCapture`
- **冒泡阶段监听器**：处理所有 React 的 `onClick`

## 完整执行流程

当点击 `#child` 时，浏览器原生捕获阶段从上往下走：

```
捕获阶段 (从上到下)：
  document  →  html  →  body  →  #root  →  #parent  →  #child
                                   ↑
                              React 的捕获监听器在这里触发！
                              此时 React 收集 Virtual DOM 树中
                              从根到目标的所有 onClickCapture，
                              按序执行：父捕获 → 子捕获
```

| 阶段 | 到达节点 | 触发 |
|------|----------|------|
| 捕获 | `#root` | **React-父捕获 → React-子捕获** |
| 捕获 | `#parent` | 原生-父捕获 |
| 捕获 | `#child` | 原生-子捕获 |
| 冒泡 | `#child` | 原生-子冒泡 |
| 冒泡 | `#parent` | 原生-父冒泡 |
| 冒泡 | `#root` | **React-子冒泡 → React-父冒泡** |

## 对比 React 16

React 16 把监听器挂在 `document` 上，而 `document` 在 DOM 树的最顶端：

```
React 16 的捕获阶段：
  document → ... → #parent → #child
      ↑
   React 捕获监听器在这里，但 document 是最后的捕获节点！
```

等等，其实 `document` 也是捕获阶段最先到达的节点。让我纠正一下：

实际上 React 16 也是把捕获监听器挂在 `document` 上，但问题在于 React 16 所有事件监听器（包括捕获）都注册在**冒泡阶段**（这是 React 16 的一个实现细节）。React 16 的合成事件捕获是通过在冒泡阶段手动模拟的，所以 React 16 的捕获事件**一定在原生冒泡之后**才执行。

而 **React 17 真正使用了原生的捕获阶段**来注册 `onClickCapture`，所以它能和原生捕获事件在同一个阶段交错执行——谁在 DOM 树上更靠上，谁就先触发。因为 `#root` 在 `#parent` 之上，所以 React 捕获先于原生父/子捕获。

## 总结

React 17 捕获提前的根本原因：

1. **React 17 真正使用了原生捕获阶段** 来注册合成捕获事件（`addEventListener(type, handler, true)`），不再像 React 16 那样在冒泡阶段模拟捕获。
2. **`#root` 在 DOM 树中位于 `#parent` 和 `#child` 之上**，捕获阶段从上往下，所以 `#root` 上的 React 捕获监听器先于 `#parent`/`#child` 上的原生捕获监听器触发。

这就是为什么你看到 React 捕获事件跑在了原生捕获事件的前面。
