## 📋 核心执行顺序规则

### 基本原则
```
原生捕获 → 原生冒泡 → React捕获 → React冒泡 
```

**关键原因**：React的所有事件（包括捕获和冒泡）都是在原生事件冒泡到根节点后才开始处理的！

---

## 🧪 测试场景1：父子嵌套 + 完整事件流

### 代码示例
```javascript
import React, { useEffect } from 'react';

function App() {
  useEffect(() => {
    // 原生事件绑定
    const parent = document.getElementById('parent');
    const child = document.getElementById('child');

    // 原生捕获
    parent.addEventListener('click', () => {
      console.log('原生-父捕获');
    }, true);

    child.addEventListener('click', () => {
      console.log('原生-子捕获');
    }, true);

    // 原生冒泡
    parent.addEventListener('click', () => {
      console.log('原生-父冒泡');
    }, false);

    child.addEventListener('click', () => {
      console.log('原生-子冒泡');
    }, false);
  }, []);

  return (
    <div id="parent" 
         onClickCapture={() => console.log('React-父捕获')}
         onClick={() => console.log('React-父冒泡')}>
      父元素
      <div id="child" 
           onClickCapture={() => console.log('React-子捕获')}
           onClick={() => console.log('React-子冒泡')}>
        子元素
      </div>
    </div>
  );
}
```

### 点击子元素时的执行顺序
```
原生-父捕获
原生-子捕获
原生-子冒泡
原生-父冒泡

原生-父捕获
原生-子捕获
原生-子冒泡
原生-父冒泡
React-父捕获
React-子捕获
React-子冒泡
React-父冒泡
```

**注意**：React 17+ 版本中，React事件会插入到原生冒泡之前！

## 🧪 测试场景2：React 16 vs React 17+ 的差异

### React 16 的执行顺序（根节点在document）
```javascript
// 点击子元素
原生-父捕获
原生-子捕获
原生-子冒泡
原生-父冒泡
React-父捕获
React-子捕获
React-子冒泡
React-父冒泡
```

### React 17+ 的执行顺序（根节点在root container）
```javascript
// 点击子元素
原生-父捕获
原生-子捕获
原生-子冒泡
React-父捕获
React-子捕获
React-子冒泡
React-父冒泡
原生-父冒泡  // 最后执行
```

**关键区别**：React 17+ 将事件委托从 `document` 移到了 root 容器，所以 React 事件会在原生冒泡到达 root 容器时执行，而不是等到冒泡到 document！

## 🧪 测试场景4：阻止冒泡的影响

### 代码示例
```javascript
import React, { useEffect } from 'react';

function App() {
  useEffect(() => {
    const parent = document.getElementById('parent');
    const child = document.getElementById('child');

    parent.addEventListener('click', (e) => {
      console.log('原生-父捕获');
    }, true);

    child.addEventListener('click', (e) => {
      console.log('原生-子捕获');
      e.stopPropagation();  // 阻止原生冒泡
    }, true);

    parent.addEventListener('click', (e) => {
      console.log('原生-父冒泡 - 不会执行');
    }, false);
  }, []);

  return (
    <div id="parent" onClick={() => console.log('React-父冒泡')}>
      父元素
      <div id="child" onClick={() => console.log('React-子冒泡')}>
        子元素
      </div>
    </div>
  );
}
```

### 执行结果
```
原生-父捕获
原生-子捕获
// 原生冒泡被阻止，不会执行
// React事件也不会执行，因为原生事件没冒泡到根节点
```

**关键点**：如果在原生捕获阶段调用 `stopPropagation()`，React事件将完全不会触发！

## 📊 完整对比表格

| 场景 | React 16 顺序 | React 17+ 顺序 |
|------|--------------|---------------|
| **原生捕获 + React捕获** | 原生捕获 → React捕获 | 原生捕获 → React捕获 |
| **React捕获 + React冒泡** | React捕获 → React冒泡 | React捕获 → React冒泡 |
| **React冒泡 + 原生冒泡** | React冒泡 → 原生冒泡 | React冒泡 → 原生冒泡 |
| **完整流程（点击子元素）** | 原生捕获 → 原生冒泡 → React捕获 → React冒泡 | 原生捕获 → React捕获 → React冒泡 → 原生冒泡 |

## 💡 关键结论

1. **原生捕获永远最先执行**：因为它是浏览器原生的事件机制
2. **React 17+ 改变了事件委托位置**：从 `document` 改为 root 容器
3. **阻止冒泡的影响范围**：
   - 原生 `stopPropagation()` 会阻止React事件
   - React `stopPropagation()` 只阻止React内部的冒泡
4. **事件委托的性能优势**：React只在根节点绑定事件，避免大量原生监听器
