# 虚拟列表深度解析

> 对应考察维度：**专业能力 → 前端工程化与性能优化 → 是否了解虚拟列表等深度优化方案**
> 字节/抖音/头条核心业务（信息流、长列表）必考题目

---

## 一、什么是虚拟列表？为什么需要？

### 问题

**面试高频问法：** "列表有 10 万条数据，怎么渲染不卡？"

**核心矛盾：**
- 一次性渲染 10 万条 DOM 节点 → 浏览器卡死、内存爆炸
- 用户只能看到可视区域内的几十条 → 其他 99.9% 的 DOM 纯属浪费

**虚拟列表思路：** 只渲染可视区域内的 DOM，其余用空白占位，滚动时动态替换可视内容。

## 二、定高虚拟列表实现（完整可运行版）

### 核心计算

```
已知条件：
  containerHeight = 600px   （容器高度）
  itemHeight = 50px         （每项固定高度）
  scrollTop = 容器当前滚动距离

计算：
  startIndex = Math.floor(scrollTop / itemHeight)
  visibleCount = Math.ceil(containerHeight / itemHeight)
  endIndex = startIndex + visibleCount
  offsetY = startIndex * itemHeight

渲染：
  visibleData = allData.slice(startIndex, endIndex + bufferCount)
  <div style="height: totalHeight">
    <div style="transform: translateY(offsetY)">
      {visibleData.map(renderItem)}
    </div>
  </div>
```

### 完整实现代码

```tsx
import React, { useState, useRef, useCallback, useEffect } from 'react';

interface VirtualListProps<T> {
  data: T[];                          // 全量数据
  itemHeight: number;                 // 单项固定高度
  containerHeight: number;            // 容器高度
  overscan?: number;                  // 缓冲区数量，默认 3
  renderItem: (item: T, index: number) => React.ReactNode;
  getItemKey?: (item: T, index: number) => string | number;
}

function VirtualList<T>({
  data,
  itemHeight,
  containerHeight,
  overscan = 3,
  renderItem,
  getItemKey,
}: VirtualListProps<T>) {
  const [scrollTop, setScrollTop] = useState(0);
  const containerRef = useRef<HTMLDivElement>(null);

  // totalHeight 用于撑开滚动条
  const totalHeight = data.length * itemHeight;

  // 可视区域能容纳的 item 数量
  const visibleCount = Math.ceil(containerHeight / itemHeight);

  // 计算起止索引
  const startIndex = Math.max(0, Math.floor(scrollTop / itemHeight) - overscan);
  const endIndex = Math.min(
    data.length,
    Math.ceil((scrollTop + containerHeight) / itemHeight) + overscan
  );

  // 可视数据切片
  const visibleData = data.slice(startIndex, endIndex);

  // 偏移量：让可视数据出现在正确的滚动位置
  const offsetY = startIndex * itemHeight;

  // 滚动处理（用 RAF 节流，避免高频触发）
  const rafRef = useRef<number>(0);
  const handleScroll = useCallback(() => {
    if (rafRef.current) cancelAnimationFrame(rafRef.current);
    rafRef.current = requestAnimationFrame(() => {
      if (containerRef.current) {
        setScrollTop(containerRef.current.scrollTop);
      }
    });
  }, []);

  // 清理 RAF
  useEffect(() => {
    return () => {
      if (rafRef.current) cancelAnimationFrame(rafRef.current);
    };
  }, []);

  return (
    <div
      ref={containerRef}
      onScroll={handleScroll}
      style={{
        height: containerHeight,
        overflow: 'auto',
        position: 'relative',
        willChange: 'transform',  // 提示浏览器优化
      }}
    >
      {/* 占位层：撑开滚动条 */}
      <div style={{ height: totalHeight, position: 'relative' }}>
        {/* 可视内容层：偏移到正确位置 */}
        <div
          style={{
            position: 'absolute',
            top: 0,
            left: 0,
            right: 0,
            transform: `translateY(${offsetY}px)`,
          }}
        >
          {visibleData.map((item, i) => {
            const actualIndex = startIndex + i;
            return (
              <div
                key={getItemKey ? getItemKey(item, actualIndex) : actualIndex}
                style={{ height: itemHeight }}
              >
                {renderItem(item, actualIndex)}
              </div>
            );
          })}
        </div>
      </div>
    </div>
  );
}

export default VirtualList;
```

### 使用示例

```tsx
function App() {
  // 模拟 10 万条数据
  const data = Array.from({ length: 100000 }, (_, i) => ({
    id: i,
    name: `用户 ${i}`,
    email: `user${i}@example.com`,
  }));

  return (
    <VirtualList
      data={data}
      itemHeight={60}
      containerHeight={600}
      overscan={5}
      renderItem={(item) => (
        <div style={{ padding: '10px', borderBottom: '1px solid #eee' }}>
          <div>{item.name}</div>
          <div style={{ color: '#999', fontSize: 12 }}>{item.email}</div>
        </div>
      )}
      getItemKey={(item) => item.id}
    />
  );
}
```

## 三、不定高虚拟列表

### 核心挑战

真实场景中列表项高度往往不一致（文字折行、图片撑高、展开收起），**无法用 `startIndex * fixedHeight` 计算偏移量**。

### 方案一：预估高度 + 实测校准（推荐）

```tsx
function DynamicVirtualList<T>({
  data,
  estimatedItemHeight = 80,  // 预估高度
  containerHeight,
  overscan = 3,
  renderItem,
  getItemKey,
}: {
  data: T[];
  estimatedItemHeight?: number;
  containerHeight: number;
  overscan?: number;
  renderItem: (item: T, index: number) => React.ReactNode;
  getItemKey?: (item: T, index: number) => string | number;
}) {
  const [scrollTop, setScrollTop] = useState(0);
  const containerRef = useRef<HTMLDivElement>(null);

  // 缓存每个 item 的实际高度和偏移量
  const itemPositions = useRef<{ height: number; offset: number }[]>([]);

  // 初始化位置缓存（使用预估高度）
  useEffect(() => {
    itemPositions.current = data.map((_, i) => ({
      height: estimatedItemHeight,
      offset: i * estimatedItemHeight,
    }));
  }, [data.length, estimatedItemHeight]);

  // 更新实际高度并重新计算偏移量
  const updateItemHeight = useCallback((index: number, actualHeight: number) => {
    const positions = itemPositions.current;
    const oldHeight = positions[index].height;
    const diff = actualHeight - oldHeight;

    if (diff !== 0) {
      positions[index].height = actualHeight;
      // 更新后续所有 item 的偏移量
      for (let i = index + 1; i < positions.length; i++) {
        positions[i].offset += diff;
      }
    }
  }, []);

  // 二分查找：根据 scrollTop 找到 startIndex
  const findStartIndex = useCallback((scrollTop: number): number => {
    const positions = itemPositions.current;
    let low = 0;
    let high = positions.length - 1;

    while (low <= high) {
      const mid = Math.floor((low + high) / 2);
      const midOffset = positions[mid].offset;

      if (midOffset === scrollTop) return mid;
      if (midOffset < scrollTop) {
        low = mid + 1;
      } else {
        high = mid - 1;
      }
    }
    return Math.max(0, low - 1);
  }, []);

  // 找到结束索引
  const findEndIndex = useCallback((startIndex: number, scrollTop: number): number => {
    const positions = itemPositions.current;
    const targetBottom = scrollTop + containerHeight;
    const overscanBottom = targetBottom + estimatedItemHeight * overscan;

    let offset = positions[startIndex].offset;
    let index = startIndex;

    while (index < positions.length && offset < overscanBottom) {
      offset += positions[index].height;
      index++;
    }
    return index;
  }, [containerHeight, estimatedItemHeight, overscan]);

  // 计算可视数据
  const startIndex = findStartIndex(scrollTop);
  const endIndex = findEndIndex(startIndex, scrollTop);
  const visibleData = data.slice(startIndex, Math.min(endIndex + 1, data.length));

  // 总高度
  const totalHeight = itemPositions.current.length > 0
    ? itemPositions.current[itemPositions.current.length - 1].offset +
      itemPositions.current[itemPositions.current.length - 1].height
    : 0;

  // 当前偏移量
  const offsetY = itemPositions.current[startIndex]?.offset ?? 0;

  const rafRef = useRef<number>(0);
  const handleScroll = useCallback(() => {
    if (rafRef.current) cancelAnimationFrame(rafRef.current);
    rafRef.current = requestAnimationFrame(() => {
      if (containerRef.current) {
        setScrollTop(containerRef.current.scrollTop);
      }
    });
  }, []);

  return (
    <div
      ref={containerRef}
      onScroll={handleScroll}
      style={{ height: containerHeight, overflow: 'auto', position: 'relative' }}
    >
      <div style={{ height: totalHeight, position: 'relative' }}>
        <div style={{ position: 'absolute', top: 0, left: 0, right: 0, transform: `translateY(${offsetY}px)` }}>
          {visibleData.map((item, i) => {
            const actualIndex = startIndex + i;
            return (
              <DynamicItem
                key={getItemKey ? getItemKey(item, actualIndex) : actualIndex}
                index={actualIndex}
                onHeightChange={updateItemHeight}
              >
                {renderItem(item, actualIndex)}
              </DynamicItem>
            );
          })}
        </div>
      </div>
    </div>
  );
}

// 测量实际高度的子组件
function DynamicItem({
  index,
  onHeightChange,
  children,
}: {
  index: number;
  onHeightChange: (index: number, height: number) => void;
  children: React.ReactNode;
}) {
  const ref = useRef<HTMLDivElement>(null);

  useEffect(() => {
    if (ref.current) {
      const height = ref.current.getBoundingClientRect().height;
      onHeightChange(index, height);
    }
  });

  return <div ref={ref}>{children}</div>;
}
```

### 方案二：ResizeObserver 动态监听

```tsx
// 用 ResizeObserver 替代 getBoundingClientRect，自动响应高度变化
function DynamicItemObserver({
  index,
  onHeightChange,
  children,
}: {
  index: number;
  onHeightChange: (index: number, height: number) => void;
  children: React.ReactNode;
}) {
  const ref = useRef<HTMLDivElement>(null);

  useEffect(() => {
    if (!ref.current) return;

    const observer = new ResizeObserver((entries) => {
      for (const entry of entries) {
        const height = entry.contentRect.height;
        onHeightChange(index, height);
      }
    });

    observer.observe(ref.current);
    return () => observer.disconnect();
  }, [index, onHeightChange]);

  return <div ref={ref}>{children}</div>;
}
```

## 四、面试追问与进阶

### Q1: scroll 事件为什么不直接用，要用 RAF 包裹？

```
原因：scroll 事件触发频率极高（某些浏览器可达 100+ 次/秒）
直接 setState  → 高频 re-render → 反而比不优化还卡

RAF（requestAnimationFrame）的作用：
- 将多次 scroll 事件合并到一帧（16.6ms）内处理
- 在浏览器绘制前执行，避免多余的计算和 DOM 操作
- 配合 will-change: transform 实现 GPU 加速
```

### Q2: 白屏问题怎么解决？

```
原因：用户快速滚动时，新区域的 DOM 还没渲染出来

解决方案：
1. overscan（缓冲区）：在可视区外多渲染 3~5 个 item
   startIndex -= overscan → 上方预渲染
   endIndex += overscan → 下方预渲染（代价：多几个 DOM 节点）

2. 骨架屏占位：对未渲染区域给灰色占位块，提升感知体验

3. 减小 itemHeight 粒度：
   如果列表项本身很高（200px+），缓冲区要相应增大
```

### Q3: 虚拟列表中如何实现"滚动到指定项"？

```tsx
// 定高场景：直接算偏移
function scrollToIndex(index: number, itemHeight: number) {
  containerRef.current?.scrollTo({
    top: index * itemHeight,
    behavior: 'smooth',
  });
}

// 不定高场景：用缓存位置
function scrollToIndexDynamic(index: number) {
  const targetOffset = itemPositions.current[index]?.offset ?? 0;
  containerRef.current?.scrollTo({
    top: targetOffset,
    behavior: 'smooth',
  });
}
```

### Q4: 虚拟列表和 IntersectionObserver 懒加载有什么区别？

| 维度 | 虚拟列表 | IntersectionObserver 懒加载 |
|------|---------|---------------------------|
| 核心思路 | 渲染可见 + 移除不可见 | 逐步渲染，不删除已渲染的 |
| DOM 总量 | 恒定（可视区 + buffer） | 随时间增长 |
| 内存 | 稳定 | 持续增长 |
| 适用场景 | 一次性加载的大量数据（10w 条） | 无限滚动（逐步加载更多） |
| 滚动条 | 真实比例 | 越来越短（因为总量在变） |

**实际项目中两者可以组合：** 虚拟列表负责渲染性能，IntersectionObserver 负责分页触发加载更多。

### Q5: 虚拟列表对 SEO 和可访问性有什么影响？

```
SEO 影响：
- 未渲染的内容不存在于 DOM 中 → 搜索引擎爬虫看不到
- 解决方案：SSR 首屏内容 + 客户端接管虚拟化

可访问性：
- 屏幕阅读器无法读到未渲染的 DOM 节点
- 解决方案：aria-rowcount / aria-rowindex 属性声明虚拟总量
```

| 库 | 特点 | 实现要点 |
|----|------|----------|
| **react-window** | 轻量（3KB），定高场景首选 | 预估高度后实测校准 |
