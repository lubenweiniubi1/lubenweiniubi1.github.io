## 面试流程预估

```
自我介绍（2-3 分钟）
├── 姓名 + 工作年限 + 当前职责
├── 核心技能（React + TypeScript + 工程化 + AI 辅助开发）
└── 一句话亮点项目

项目深挖（10-15 分钟）
├── "选一个最有代表性的项目详细讲讲"
├── 追问：技术难点、你的角色、量化成果
└── 策略：用 STAR 法则 + 主动带出技术深度

React 核心原理（10-15 分钟）
├── Fiber 架构 → Diff 算法 → Hooks 陷阱
├── React 19/RSC 新特性了解
└── 策略：每个概念都有白板讲解能力

工程化与性能（10-15 分钟）
├── Lint 工具链 → 性能优化实战
├── 虚拟列表 / 防抖节流 手写
└── 策略：讲原理 + 讲落地 + 讲数据

编码能力（10 分钟）
├── 手写题 1：Promise 调度器 / LRU Cache（大概率）
├── 手写题 2：深拷贝 / 防抖节流 / 数组转树
└── 策略：平时多手写，面试时边说边写

反问环节（3-5 分钟）
├── "团队目前的技术栈和项目情况？"
├── "这个岗位对 AI Coding 的接受度如何？"
└── 策略：问得具体，展示你已经在思考如何融入团队
```

## 高频 TOP 10 问题速记

| 排名 | 问题 | 一句话答案 |
|------|------|-----------|
| 1 | Fiber 解决了什么问题？ | 将递归渲染改为可中断的异步渲染，主线程不卡死 |
| 2 | Hooks 闭包陷阱怎么解？ | useRef 保存最新值 + 正确的依赖声明 |
| 3 | 虚拟列表原理？ | 只渲染可视区 DOM + padding 占位模拟滚动 |
| 4 | 防抖节流区别？ | 防抖=停后再执行，节流=固定间隔执行 |
| 5 | Table 怎么封装？ | 三层：核心层+业务层+样式层，配置化驱动 |
| 6 | 性能优化做过哪些？ | 分步加载+虚拟列表+缓存+代码分割 |
| 7 | AI Coding 怎么用？ | 样板代码生成+类型定义+测试用例+代码审查辅助 |
| 8 | Diff 算法的三个假设？ | 类型不同重建+key相同复用+同层比较 |
| 9 | useMemo 什么时候用？ | 昂贵计算+作为其他 hook 的依赖 |
| 10 | Promise 调度器怎么写？ | 维护 running 计数 + queue，每个任务结束后递归调用 run |

## 最后提醒

1. **每道题都带入一个真实案例**——不只是背答案
2. **先说结论再展开**——AI 面试官分析语音时先听到关键词
3. **停顿不可怕，关键停顿后有清晰表达**——宁可慢一点
4. **主动展示 AI 工具经验**——这是字节 OD 目前特别关注的维度
5. **手写题边说边写**——展示思考过程也是评分点
