## 1. 架构概览

```
ui-cross-platform/
├── packages/
│   ├── components/    ← 组件库（Button、Input ...）
│   │   └── src/
│   │       ├── index.ts           # 统一导出入口
│   │       └── Button/
│   │           ├── Button.tsx      # 组件实现
│   │           ├── index.scss      # 组件样式（CSS Variables 驱动）
│   │           ├── types.ts        # 类型定义 + 导出
│   │           ├── constants.ts    # 常量 + 联合类型
│   │           ├── index.tsx       # Barrel re-export
│   │           └── adapter/        # 平台差异适配
│   │               ├── weapp.ts    # 微信小程序特有逻辑
│   │               ├── tt.ts       # 抖音小程序特有逻辑
│   │               └── h5.ts       # H5 特有逻辑
│   │
│   ├── styles/        ← 设计 Token + Mixin + 主题
│   │   └── src/
│   │       ├── tokens/             # SCSS 变量（$color-primary ...）
│   │       ├── mixins/             # 公共样式片段（flex-center ...）
│   │       └── themes/             # CSS Variables（:root 声明）
│   │
│   ├── utils/         ← 工具函数（平台检测、单位转换）
│   │   └── src/
│   │       ├── platform.ts         # isWeapp / isTT / isH5 / getPlatform
│   │       └── unit.ts             # pxToRpx / rpxToPx / getPixelRatio
│   │
│   ├── demo/          ← Taro 应用（三端真机预览）
│   │   ├── config/index.ts         # Taro 构建配置 + alias + compile.include
│   │   └── src/pages/index/        # 组件预览页
│   │
│   └── storybook/     ← Storybook（隔离开发 + 文档）
│       ├── .storybook/main.js      # webpack 配置 + alias + SCSS
│       └── stories/                # *.stories.tsx
```

### 核心设计决策

| 决策 | 原因 |
|------|------|
| **Demo 不依赖 components build** | alias 直接指向 `components/src`，改源码自动热更新 |
| **Storybook 不依赖 Taro** | Button.tsx 做了 H5 降级（`TaroButton \|\| 'button'`），纯 React 环境可渲染 |
| **CSS Variables 驱动样式** | 主题切换只需换 `data-theme`，不重新编译 |
| **Vite 构建合一带货** | utils 源码打进 components 产物，消费者只装一个包 |
| **Adapter 模式** | 平台差异隔离在 `adapter/` 目录，组件本体不感知平台 |

## 2. 开发闭环

### 2.1 新建组件

以 Button 为模板，新建一个 `Input` 组件：

```
packages/components/src/Input/
├── Input.tsx         ← forwardRef + classNames + adapter 调用
├── index.scss        ← @import '~@ui-cross-platform/styles'; BEM 命名
├── types.ts          ← InputProps, InputInstance 接口
├── constants.ts      ← INPUT_DEFAULT_PROPS, INPUT_TYPES 常量
├── index.tsx         ← export { Input }; + re-export types
└── adapter/          ← 如有平台差异，复制 Button 的 adapter 结构
    ├── weapp.ts
    ├── tt.ts
    └── h5.ts
```

**组件入口注册**：[packages/components/src/index.ts](packages/components/src/index.ts)

```ts
// 从子目录显式导入再导出（避免 barrel re-export 的 Babel bug）
import { Input } from './Input';
export { Input };
export type { InputProps, InputInstance } from './Input';
```

### 2.2 Storybook 预览（组件开发主力）

**适用场景**：开发组件本身 —— 调整样式、调试交互、看 Controls 面板参数变化。

```bash
cd packages/storybook
yarn dev          # 打开 http://localhost:6006
```

**新建 Story**：[packages/storybook/stories/Input.stories.tsx](packages/storybook/stories/Button.stories.tsx)

```tsx
import { Meta, StoryObj } from '@storybook/react';
import { Input } from '@ui-cross-platform/components';

const meta: Meta<typeof Input> = {
  title: '表单/Input',
  component: Input,
  args: { placeholder: '请输入...', disabled: false },
  argTypes: {
    type: { control: 'select', options: ['text', 'password', 'number'] },
  },
};
export default meta;
type Story = StoryObj<typeof Input>;

export const Default: Story = {};
export const Disabled: Story = { args: { disabled: true } };
```

**Storybook 不需要 Taro 运行时**：Button.tsx 中的降级逻辑确保了在 `@tarojs/components` 不可用时回退到原生 `<button>`：

```ts
// packages/components/src/Button/Button.tsx
const ButtonElement: any = typeof TaroButton === 'function' ? TaroButton : 'button';
```

**Storybook webpack 配置**（[main.js](packages/storybook/.storybook/main.js)）做了三件事：

1. 新增 `babel-loader` 规则编译 `packages/` 下的 `.tsx`
2. 新增 `sass-loader` 规则编译 SCSS
3. 设置 alias 指向各包的 `src/`

### 2.3 Demo 三端验证

**适用场景**：组件在真实小程序 / H5 环境中的行为验证（尤其是 openType、登录授权等平台 API）。

```bash
cd packages/demo

# H5 浏览器（最常用）
yarn dev:h5          # → http://localhost:10086

# 微信小程序（需先打开微信开发者工具）
yarn dev:weapp       # → 自动打开微信开发者工具

# 抖音小程序
yarn dev:tt          # → 自动打开抖音开发者工具
```

**前提条件**：demo 的 `devDependencies` 需包含对应平台插件：

```json
{
  "devDependencies": {
    "@tarojs/plugin-platform-weapp": "^3.6.40",  // → dev:weapp
    "@tarojs/plugin-platform-tt": "^3.6.40",     // → dev:tt
    "@tarojs/plugin-platform-h5": "^3.6.40"      // → dev:h5 （当前缺失！）
  }
}
```

**Demo 如何引用组件源码**（[config/index.ts](packages/demo/config/index.ts)）：

```ts
// alias: 将包名直接映射到源码目录
alias: {
  '@ui-cross-platform/components': path.resolve(__dirname, '../../components/src'),
  '@ui-cross-platform/utils': path.resolve(__dirname, '../../utils/src'),
},

// compile.include: 让 Taro webpack 编译这些外部目录
compile: {
  include: [
    path.resolve(__dirname, '../../components'),
    path.resolve(__dirname, '../../utils'),
    path.resolve(__dirname, '../../styles'),
  ],
},
```

这意味着 **改一行组件源码 → Demo 热更新，不需要重新 build**。

**添加新的预览页**：

1. [packages/demo/src/pages/input/index.tsx](packages/demo/src/pages/input/index.tsx) — 页面组件
2. [packages/demo/src/pages/input/index.config.ts](packages/demo/src/pages/input/index.config.ts) — 页面配置
3. [packages/demo/src/app.config.ts](packages/demo/src/app.config.ts) — 在 `pages` 数组中注册路径

### 2.4 构建

#### 构建 components

```bash
cd packages/components
yarn build
```

产物（[vite.config.ts](packages/components/vite.config.ts)）：

```
dist/
├── index.js         ← CJS（main 入口）
├── index.esm.js     ← ESM（module 入口）
├── index.d.ts       ← 类型声明（vite-plugin-dts 生成）
└── style.css        ← 所有组件样式合并单文件
```

关键构建策略：

- `utils` 源码**打包进** components（不 externalize）→ 消费者只装一个包
- `react`、`@tarojs/*`、`classnames` → external，由消费者提供
- SCSS → 消费者自行编译（`~@ui-cross-platform/styles` 走 sass-loader）

#### 构建其他包

```bash
# utils（如有改动）
cd packages/utils && yarn build

# styles 不需要 build —— src/ 直接作为 npm 入口发布
```

## 3. 跨平台适配

### Adapter 模式

每个组件在 `adapter/` 目录下为三个平台分别维护适配逻辑：

```
Button/adapter/
├── weapp.ts    ← getWeappButtonProps() + isWeapp()
├── tt.ts       ← getTtButtonProps() + isTt()
└── h5.ts       ← getH5ButtonProps() + isH5()
```

**组件内部调用方式**（Button.tsx）：

```tsx
import { getWeappButtonProps, isWeapp } from './adapter/weapp';
import { getH5ButtonProps, isH5 } from './adapter/h5';
import { getTtButtonProps, isTt } from './adapter/tt';

const platformProps = useMemo(() => {
  if (isWeapp())  return getWeappButtonProps(props);
  if (isTt())     return getTtButtonProps(props);
  if (isH5())     return getH5ButtonProps(props);
  return {};
}, [props]);

// 合并到组件根元素
<ButtonElement {...platformProps} {...restProps}>
```

### 适配了什么？

以 Button 为例：

| 差异点 | 微信小程序 | 抖音小程序 | H5 |
|--------|-----------|-----------|-----|
| openType | `openType` 属性 + `onGetUserInfo` 等事件 | `openType` 属性 | 不支持（原生 `<button>` 无此概念） |
| loading | 原生 `loading` 属性 | 原生 `loading` 属性 | CSS class 模拟 |
| size | `mini` | 无映射 | CSS class 控制 |
| htmlType | 不支持 | 不支持 | `type="submit"` 等 |

### 平台检测

`utils/platform.ts` 封装 `Taro.getEnv()`，带缓存和降级：

```ts
// H5 降级：Storybook / 单测中 Taro 未初始化，默认视为 H5
function getEnv(): string {
  try {
    return require('@tarojs/taro').getEnv();
  } catch {
    return 'WEB';   // 兜底
  }
}
```

## 4. 消费方式

### 4.1 Monorepo 内部引用（开发时）

**Demo 和 Storybook** 通过 **路径 alias** 直接引用源码：

```ts
// webpack / vite resolve.alias
'@ui-cross-platform/components' → '../../components/src'
'@ui-cross-platform/utils'       → '../../utils/src'
'@ui-cross-platform/styles'      → '../../styles/src'
```

SCSS 引用同理，由构建工具（webpack sass-loader / Vite）编译：

```scss
@import '~@ui-cross-platform/styles';   // 获取所有 token + mixin
```

优点：零构建成本，改源码即时生效。

### 4.2 外部 npm 引用（发布后）

外部 Taro 项目安装：

```bash
npm install @ui-cross-platform/components
```

**使用方法**：

```tsx
// 1. 引入组件
import { Button } from '@ui-cross-platform/components';
import type { ButtonProps } from '@ui-cross-platform/components';

// 2. 引入样式（CSS Variables 主题）
// 在 app.ts 或全局入口执行一次：
import '@ui-cross-platform/components/dist/style.css';

// 也可以只引入主题变量（如果自建样式体系）：
// 在 app.scss 中：
// @import '~@ui-cross-platform/styles/src/themes/index';
```

**外部项目 Taro 配置**：需要将 `@ui-cross-platform` 相关包加入 webpack 编译范围（因为 node_modules 中默认不编译）：

```ts
// 外部项目的 config/index.ts
compile: {
  include: [
    path.resolve(__dirname, 'node_modules/@ui-cross-platform/components'),
    path.resolve(__dirname, 'node_modules/@ui-cross-platform/utils'),
  ],
},
```

## 5. 版本发布

### 5.1 版本策略

项目使用 **Lerna independent 模式**（[lerna.json](lerna.json)），每个可发布包**独立版本号**：

| 包 | npm 名 | 版本演进 |
|----|--------|---------|
| components | `@ui-cross-platform/components` | 独立 |
| styles | `@ui-cross-platform/styles` | 独立 |
| utils | `@ui-cross-platform/utils` | 独立 |
| demo | 无（private） | 不发布 |
| storybook | 无（private） | 不发布 |

这意味着改一个 Button 的样式，只需升 `components` 的版本号，`styles` 和 `utils` 不受影响。

**语义化版本（SemVer）**：

| 变更类型 | 版本号 | 示例 |
|---------|--------|------|
| Patch | `0.0.1` → `0.0.2` | Bug 修复、样式微调 |
| Minor | `0.0.1` → `0.1.0` | 新增组件、新增 prop（向后兼容） |
| Major | `0.0.1` → `1.0.0` | 移除组件、prop 重命名、Breaking Change |

**在 `0.x` 阶段**，Minor 可以包含 Breaking Change（遵循 SemVer 惯例：0.x 版本不稳定）。

### 5.2 发版流程

**Step 1 — 确认所有改动已提交**

```bash
git status          # 确保没有未提交的改动
```

**Step 2 — 构建所有可发布包**

```bash
# 在根目录执行全量构建
yarn build          # = lerna run build，会构建 components + utils
```

构建完成后验证产物：

```bash
ls packages/components/dist/
# 应该看到: index.js  index.esm.js  index.d.ts  style.css

ls packages/utils/dist/
# 应该看到: index.js  index.esm.js  index.d.ts
```

**Step 3 — Lerna 升级版本号**

```bash
# 预览：看看哪些包会升版本，不会真的改
npx lerna version --no-private --no-git-tag-version --no-push

# 确认无误后正式执行：
npx lerna version --no-private
```

`lerna version` 会做三件事：
1. 检测自上次 tag 以来哪些包有改动
2. 交互式询问每个包的版本号（patch / minor / major / 手动输入）
3. 写入 `package.json` 的 `version` 字段
4. 创建 git commit + tag（如 `@ui-cross-platform/components@0.1.0`）
5. 推送到 remote

**如果不想交互**，可以预设版本：

```bash
npx lerna version patch --no-private    # 所有改动包统一升 patch
npx lerna version minor --no-private    # 所有改动包统一升 minor
```

**Step 4 — 发布到 npm**

```bash
# 发布所有有改动的包
npx lerna publish from-git

# 如果是首次发布（scoped package 默认 private）：
npx lerna publish from-git --access public
```

`from-git` 的含义：根据 git tag 决定发布哪些包（即 Step 3 创建的 tag），而不是重新询问版本号。

**Step 5 — 验证发布结果**

```bash
npm view @ui-cross-platform/components version
npm view @ui-cross-platform/components dist-tags
```

### 5.3 单包独立发布

如果只改了 `components`，不需要发布 `styles` 和 `utils`：

```bash
# 方式 A：lerna version 只升有改动的包
npx lerna version --no-private
# Lerna 会自动检测改动范围，没改的包不会提示升级

# 方式 B：直接在子包中手动操作
cd packages/components
npm version patch           # 0.0.1 → 0.0.2
yarn build
npm publish --access public
git push --follow-tags      # 把 tag 推到 remote
```

> **推荐方式 A**：Lerna 自动管理 tag 命名和 commit 格式，避免手动拼错。

### 5.4 Alpha / Beta 预发布

当需要给 QA 或合作方提前测试时，发 prerelease 版本：

```bash
# 交互式，选择 prepatch / preminor / premajor
npx lerna version --no-private --preid alpha
# 示例输出: @ui-cross-platform/components@0.0.2-alpha.0
```

npm 不会把 `alpha`/`beta` 版本设为 `latest` tag，消费者需要显式安装：

```bash
npm install @ui-cross-platform/components@0.0.2-alpha.0
```

测试通过后，正式发布时去掉 preid：

```bash
npx lerna version --no-private --conventional-graduate
# 将 alpha.0 → 正式版本
```
