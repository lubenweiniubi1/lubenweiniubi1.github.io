## 一、前端工程化（ESLint / Prettier / Husky）

### Q1: 你如何在前端项目中落地工程化规范？

**标准答案框架——完整 Lint 工作流：**

```
写代码 → 保存自动格式化 → git commit 触发 Husky
→ lint-staged 仅检查暂存文件 → commitlint 校验提交信息
→ 推送 → CI 最终守门
```

**工具链组合：**

| 工具 | 作用 | 触发时机 |
|------|------|----------|
| ESLint | 代码质量检查（未使用变量、潜在 bug） | 保存时 + pre-commit |
| Prettier | 代码风格统一（缩进、引号、分号） | 保存时自动格式化 |
| Husky | 管理 Git Hooks | `git commit` 时触发 |
| lint-staged | 仅检查 git 暂存文件 | pre-commit hook 内 |
| Commitlint | 提交信息规范校验 | commit-msg hook 内 |

### Q2: ESLint 代码检查的过程是什么？

1. **解析（Parse）**：将代码解析为 AST（抽象语法树）
2. **遍历（Traverse）**：深度优先遍历 AST 节点
3. **规则匹配（Lint）**：每个规则是一个 visitor，在进入/离开特定节点类型时执行检查
4. **报告（Report）**：收集所有违规项，统一报告

**追问：ESLint 9 的 Flat Config 有什么变化？**
- 使用 `eslint.config.js` 替代 `.eslintrc`
- 不再用 `extends`，改为数组形式的 `configs`
- 插件需要通过 `import` 引入

### Q3: 如何在团队中推行 Lint 规范？

**落地策略（分阶段）：**

1. **阶段一：本地自动化**（最小阻力）
   - 配置编辑器保存时自动格式化
   - 提供统一的 VSCode 配置 `.vscode/settings.json`
2. **阶段二：提交时卡口**
   - Husky + lint-staged 在 `pre-commit` 时自动修复
   - 只检查变更文件，速度可控
3. **阶段三：CI 守门**
   - CI 中执行全量 lint 作为最终门禁
   - 不通过则不允许合并
4. **阶段四：持续优化**
   - 定期 review 规则是否合理
   - 将高频问题写入团队文档

## 二、性能优化——分步加载与资源优化

### Q4: 首屏加载优化有哪些手段？

**按加载阶段分层回答：**

```
网络层：
├── CDN 加速静态资源
├── HTTP/2 多路复用
├── 关键资源 preload / prefetch
├── 非关键资源 async / defer
└── 接口缓存（HTTP 缓存 + 内存缓存）

构建层：
├── 代码分割（路由级 + 组件级）
├── Tree Shaking
├── 图片压缩 + WebP/AVIF 格式
└── 第三方库按需引入

渲染层：
├── SSR / SSG / RSC
├── 骨架屏 / Loading 态
├── 首屏关键 CSS 内联
└── 字体加载优化（font-display: swap）
```

### Q5: 什么是分步加载？你在项目中如何实践？

**分步加载策略：**

```
第一步（Critical）：首屏可见内容 → 服务端渲染 / 内联关键 CSS
第二步（High）：首屏交互 JS → React.lazy 代码分割，preload
第三步（Medium）：非首屏路由 → 路由懒加载
第四步（Low）：图片 / 字体 / 分析脚本 → 懒加载 + async
```

**React 中实践：**
```jsx
// 路由级分步加载
const Dashboard = React.lazy(() => import('./pages/Dashboard'));

// 组件级分步加载
```

### Q6: 接口缓存策略有哪些？

**分层缓存策略：**

```js
// 1. HTTP 缓存（强缓存 + 协商缓存）
// Cache-Control: max-age=3600  // 强缓存 1 小时
// ETag / Last-Modified        // 协商缓存

// 2. 内存缓存（SWR 模式）
function useSWR(key, fetcher) {
  const cache = useRef(new Map());

  const { data, error } = useSWR(key, async (k) => {
    if (cache.current.has(k)) {
      return cache.current.get(k);  // 先返回缓存
    }
    const result = await fetcher(k);
    cache.current.set(k, result);   // 更新缓存
    return result;
  }, {
    revalidateOnFocus: true,        // 标签页聚焦时重新验证
    dedupingInterval: 2000,         // 2 秒内去重
  });

  return { data, error };
}

// 3. 请求去重
const pendingRequests = new Map();
async function fetchWithDedup(url) {
  if (pendingRequests.has(url)) {
    return pendingRequests.get(url);  // 复用进行中的请求
  }
  const promise = fetch(url).then(r => r.json());
  pendingRequests.set(url, promise);
  promise.finally(() => pendingRequests.delete(url));
  return promise;
}
```

## 三、防抖与节流（高频手写 + 场景题）

### Q7: 手写防抖函数（支持立即执行 + cancel）

```js
function debounce(fn, delay = 300, immediate = false) {
  let timer = null;

  const debounced = function (...args) {
    if (timer) clearTimeout(timer);

    if (immediate) {
      const callNow = !timer;
      timer = setTimeout(() => { timer = null; }, delay);
      if (callNow) fn.apply(this, args);
    } else {
      timer = setTimeout(() => {
        fn.apply(this, args);
        timer = null;
      }, delay);
    }
  };

  debounced.cancel = function () {
    clearTimeout(timer);
    timer = null;
  };

  return debounced;
}
```

### Q8: 手写节流函数（定时器版 + 时间戳版对比）

```js
// 时间戳版：第一次立即执行，停止后不执行最后一次
function throttle_timestamp(fn, delay = 300) {
  let lastTime = 0;
  return function (...args) {
    const now = Date.now();
    if (now - lastTime >= delay) {
      fn.apply(this, args);
      lastTime = now;
    }
  };
}

// 定时器版：第一次延迟执行，停止后会执行最后一次
function throttle_timer(fn, delay = 300) {
  let timer = null;
  return function (...args) {
    if (timer) return;
    timer = setTimeout(() => {
      fn.apply(this, args);
      timer = null;
    }, delay);
  };
}

// 综合版：第一次立即执行 + 停止后执行最后一次
function throttle(fn, delay = 300) {
  let lastTime = 0, timer = null;
  return function (...args) {
    const now = Date.now();
    const remaining = delay - (now - lastTime);

    if (remaining <= 0) {
      if (timer) { clearTimeout(timer); timer = null; }
      fn.apply(this, args);
      lastTime = now;
    } else if (!timer) {
      timer = setTimeout(() => {
        fn.apply(this, args);
        lastTime = Date.now();
        timer = null;
      }, remaining);
    }
  };
}
```

### Q9: 防抖和节流的区别？各自适用场景？

| 维度 | 防抖（Debounce） | 节流（Throttle） |
|------|-----------------|------------------|
| 行为 | 连续触发只执行最后一次（或第一次） | 固定时间间隔执行一次 |
| 适用场景 | 搜索输入联想、表单验证、resize 回调 | 滚动加载、鼠标拖拽、视频进度 |
| 抖音/头条案例 | 搜索框输入停顿后发请求 | 视频流滚动时固定频率加载下一批 |

## 四、虚拟列表（深度优化方案）

### Q10: 虚拟列表的实现原理

**核心思想**：只渲染可视区域内的 DOM 节点，用空白占位模拟滚动高度。

```
可视区域计算：
├── startIndex = Math.floor(scrollTop / itemHeight)
├── endIndex = Math.ceil((scrollTop + containerHeight) / itemHeight)
├── visibleData = allData.slice(startIndex, endIndex + overscan)
├── paddingTop = startIndex * itemHeight   （撑开上方空间）
└── paddingBottom = (total - endIndex) * itemHeight

关键事件：
├── scroll 事件 + requestAnimationFrame 节流
├── ResizeObserver 监听容器尺寸变化
└── IntersectionObserver 做懒加载
```

### Q11: 不定高列表项如何实现虚拟列表？

**两种方案：**

1. **预估高度 + 实测校准（react-window 做法）**
   ```
   - 预设一个预估高度（如 80px）
   - 实际渲染后通过 getBoundingClientRect() 获取真实高度
   - 更新高度缓存并重新计算偏移量
   ```

2. **ResizeObserver 动态监听（react-virtuoso 做法）**
   ```
   - 用 ResizeObserver 监听每个已渲染项的高度变化
   - 自动更新总高度和偏移量
   ```

### Q12: 虚拟列表的空白闪烁问题怎么解决？

- **overscan（缓冲区）**：在可视区域外多渲染 2-5 个 item
- **骨架屏占位**：给未渲染区域预置灰色骨架
- **min-height 兜底**：给每行设置最小高度，避免高度突变

## 五、其他性能优化场景题

### Q13: 如何检测和优化 Long Task？

```js
// 检测
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    if (entry.duration > 50) {
      console.warn('Long Task:', entry);
    }
  }
});
observer.observe({ entryTypes: ['longtask'] });

// 优化：拆分长任务
function splitLongTask(tasks, chunkSize = 5) {
  let index = 0;
  function processChunk() {
    const end = Math.min(index + chunkSize, tasks.length);
    for (let i = index; i < end; i++) tasks[i]();
    index = end;
    if (index < tasks.length) {
      requestIdleCallback(processChunk); // 或 setTimeout
    }
  }
  requestIdleCallback(processChunk);
}
```

### Q14: 大列表的搜索筛选怎么做性能优化？

- **前端筛选**：使用 Web Worker 在独立线程执行搜索
- **防抖输入**：输入框 debounce 300ms 后再触发搜索
- **搜索结果也走虚拟列表**：避免搜索结果 DOM 节点过多
- **高亮算法**：避免正则全局替换，用分割+插值方式
