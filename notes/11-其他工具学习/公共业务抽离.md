## 🎯 React 公共业务抽离 - 完整指南

---

## 📋 **核心原则**

### **1. 单一职责原则**
- 每个组件只负责一个特定功能
- 避免"上帝组件"（一个组件做太多事情）

### **2. 高内聚低耦合**
- **功能独立**：组件内部逻辑自成体系
- **数据隔离**：通过 props/events 与外部通信
- **样式封闭**：使用 CSS Modules 避免样式污染
## 🗣️ 最直白的解释


### **高内聚 = 一家人住一起**

- 做饭的工具都在厨房
- 洗澡的东西都在卫生间  
- 睡觉的床都在卧室

**改起来方便**：想改厨房就改厨房，不用满屋子找锅碗瓢盆

---

### **低耦合 = 邻居关系**

- 你家装修，不影响邻居家
- 邻居家吵架，不关你事
- 各过各的，互不干扰

**改起来安全**：改你家的代码，不会把别人的功能搞挂


## 🏗️ **分层架构设计**

```
┌─────────────────────────────────────┐
│         容器组件 (Container)         │
│   - 处理数据流                       │
│   - 组合业务组件                     │
│   - 连接 Redux/Context              │
└─────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────┐
│         业务组件 (Business)          │
│   - 包含业务规则                     │
│   - 组合基础组件                     │
│   - 示例：OrderTable, UserCard      │
└─────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────┐
│         基础组件 (Base/UI)           │
│   - 纯 UI 展示                       │
│   - 无业务逻辑                       │
│   - 示例：Button, Input, Modal      │
└─────────────────────────────────────┘
```

## 🔧 **主要抽离方式**

### **方式一：自定义 Hooks（推荐）**

```javascript
// 抽离登录验证逻辑
function useLoginValidation() {
  return (values) => {
    const errors = {};
    if (!values.username) errors.username = '请输入用户名';
    if (!values.password) errors.password = '请输入密码';
    return errors;
  };
}

// 抽离表单提交逻辑
function useFormSubmit(apiUrl) {
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  const submit = async (data) => {
    try {
      setLoading(true);
      setError(null);
      const response = await fetch(apiUrl, {
        method: 'POST',
        body: JSON.stringify(data),
      });
      return await response.json();
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  return { loading, error, submit };
}
```

**使用示例：**
```javascript
function LoginForm() {
  const validate = useLoginValidation();
  const { loading, error, submit } = useFormSubmit('/api/login');
  
  const handleSubmit = async (values) => {
    const errors = validate(values);
    if (Object.keys(errors).length > 0) return;
    await submit(values);
  };

  return (
    <form onSubmit={handleSubmit}>
      {/* 表单内容 */}
    </form>
  );
}
```

### **方式二：高阶组件（HOC）**

```javascript
// 包装组件 - 统一添加样式
function withTheme(Component, theme) {
  return function ThemedComponent(props) {
    return <Component {...props} style={theme} />;
  };
}

// 抽离状态管理
function withAuth(Component) {
  return function AuthComponent(props) {
    const [user, setUser] = useState(null);
    const [loading, setLoading] = useState(true);

    useEffect(() => {
      checkAuth().then(setUser).finally(() => setLoading(false));
    }, []);

    if (loading) return <Loading />;
    if (!user) return <Login />;
    
    return <Component {...props} user={user} />;
  };
}

// 使用
const ProtectedPage = withAuth(Dashboard);
```

### **方式三：Render Props**
#### 核心概念
Render Props 是一种 React 组件设计模式，其中一个组件通过接收一个函数作为 prop，然后调用这个函数来决定渲染什么内容。
#### 核心思想
一个组件负责提供数据和逻辑，但不决定如何显示，而是将渲染的控制权交给使用者。


```javascript
// 数据获取组件
function DataFetcher({ url, children }) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch(url)
      .then(res => res.json())
      .then(setData)
      .catch(setError)
      .finally(() => setLoading(false));
  }, [url]);

  return children({ data, loading, error });
}

// 使用
<DataFetcher url="/api/users">
  {({ data, loading, error }) => {
    if (loading) return <Spinner />;
    if (error) return <Error message={error} />;
    return <UserList users={data} />;
  }}
</DataFetcher>
```

### **方式四：Context API（全局状态）**

```javascript
// 创建 Context
const UserContext = createContext();

// Provider 组件
function UserProvider({ children }) {
  const [user, setUser] = useState(null);
  const [permissions, setPermissions] = useState([]);

  const login = async (credentials) => {
    const userData = await api.login(credentials);
    setUser(userData);
    setPermissions(userData.permissions);
  };

  const logout = () => {
    setUser(null);
    setPermissions([]);
  };

  return (
    <UserContext.Provider value={{ user, permissions, login, logout }}>
      {children}
    </UserContext.Provider>
  );
}

// 自定义 Hook 使用
function useUser() {
  const context = useContext(UserContext);
  if (!context) throw new Error('useUser must be used within UserProvider');
  return context;
}

// 使用
function Header() {
  const { user, logout } = useUser();
  return (
    <div>
      <span>{user?.name}</span>
      <button onClick={logout}>退出</button>
    </div>
  );
}
```

## ✅ **最佳实践总结**

| 场景 | 推荐方式 | 优点 |
|------|---------|------|
| 表单验证、数据获取 | 自定义 Hook | 逻辑复用，测试方便 |
| 权限控制、主题切换 | HOC | 代码简洁，易于组合 |
| 复杂状态管理 | Context API | 全局共享，避免 props drilling |
| 通用 UI 模式 | 组件封装 | 视觉统一，易于维护 |
| 动态渲染逻辑 | Render Props | 灵活控制，类型安全 |

## 🚀 **实际项目建议**

1. **先写后抽**：不要过早抽象，等代码重复 2-3 次后再抽离
2. **命名规范**：Hook 用 `useXxx`，HOC 用 `withXxx`，组件用 `XxxComponent`
3. **TypeScript**：为抽离的组件和 Hook 定义清晰的类型
4. **文档注释**：为每个抽离的模块添加使用说明
5. **单元测试**：为抽离的逻辑编写测试用例

这样可以让你的 React 项目更加**可维护、可扩展、易测试**！ 🎉
